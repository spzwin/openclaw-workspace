# 会议总结与待办 — 约束与模版

当用户请求「总结这场会议」「生成会议报告」或「列待办」「有哪些要做的」时，本技能按以下约束与模版输出，便于统一格式与质量。

---

# 一、会议总结报告

---

## 报告约束（必须遵守）

| 约束项 | 说明 |
|--------|------|
| **语言** | 报告语言 = 会议内容的语言（中文/英文/粤语等），禁止中途切换 |
| **依据** | 严格基于会议原文，禁止添加、推测或虚构任何信息 |
| **内容过少** | 会议内容少于 100 字或无实质内容时，只输出「无法生成有效的会议报告」，禁止虚构 |
| **格式** | 全文 Markdown；可修订明显错别字、专业术语与人名错误，不改变原意 |
| **禁止虚构** | 时间、人物、数据、决策结论均不得编造，只总结原文明确提到的内容 |

---

## 报告模版（结构固定）

1. **会议标题（必须）**  
   - 格式：`# [核心主题][会议类型]`  
   - 会议类型仅从下列选一：沟通会、讨论会、培训会、评审会、宣导会、面试、汇报会、决策会、启动会、复盘会  

2. **报告开头**  
   - 直接以段落形式开始，不加小标题  
   - 100–300 字概述：会议核心内容、主要讨论点、关键决策、重要结论  

3. **详细报告**  
   - 按会议性质组织内容，重点突出：重点结论、目标、里程碑、决策事项  
   - 禁止输出「会议性质分析」「本次会议属于XX会」「会议类型判断」等元描述  

4. **待办事项（按需）**  
   - 仅当会议内容中存在**符合标准的真实待办**时，才输出本节；否则整节不输出，也不写「无待办」  
   - 格式：
     ```markdown
     ## ✅ 待办事项
     | 事项内容 | 责任主体 | 优先级 | 时间要求 |
     |---------|---------|-------|---------|
     | 具体任务描述 | 负责人/待明确 | 高/中/低 | 截止时间/待明确 |
     ```
   - 真实待办标准：会议中明确达成共识 + 包含可执行动作（完成、编写、修复、对接、统计、汇总、评审、落实、跟进等）+ 有明确任务目标；责任主体/时间未明确则填「待明确」

---

# 二、待办事项提取

当用户请求「列一下待办」「有哪些要做的」时，按以下约束与模版输出。

## 待办提取约束（必须遵守）

| 约束项 | 说明 |
|--------|------|
| **依据** | 严格基于会议原文，禁止虚构负责人、时间、年份 |
| **宁少勿多** | 无法完全确定时不列为待办，只列明确拍板的 |
| **无待办时** | 禁止输出「该会议暂无待办事项」等否定性结论；可输出一句中性说明 |
| **单条** | 每条 30–100 字；忠实原文可轻度润色；同一任务以最后一次为准 |

## 待办提取模版

- **存在待办时**：逐条列表，格式 `- [负责人] 在 [时间] 前完成 [任务描述]`，未明确则写「待明确」。
- **不存在待办时**：仅一句中性说明，例如「经分析，本次会议未形成需列出的待办事项。」

**识别标准**（与报告中待办一致）：会议中明确达成共识 + 可执行动作（完成、编写、修复、对接、统计、汇总、评审、落实、跟进等）+ 明确任务目标。随口提及、方向性表达、已完成事项、仅描述分工无本次具体任务的，不列入。

# 三、与提示词的关系

- **报告**：实际生成使用 SKILL.md 文末「会议报告生成提示词」及 `src/prompts.py` 的 `REPORT_SYSTEM_PROMPT`。
- **待办提取**：使用 SKILL.md 文末「会议待办事项提取提示词」及 `src/prompts.py` 的 `TODO_EXTRACT_PROMPT`。

本文档为上述约束与模版的用户侧说明。
