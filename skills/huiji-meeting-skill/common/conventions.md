# 通用约束与约定

## 1. 生产域名约束

- 慧记 API：`https://sg-al-ai-voice-assistant.mediportal.com.cn`
- 禁止在发布内容中出现本地开发地址或非生产协议地址

## 2. Header 规范

- **慧记接口**：`appKey`（必传）、`Content-Type: application/json`

## 3. 输出与脱敏

- 对用户输出：结论/摘要/可访问链接/必要操作提示
- 默认不输出：`token`、`appKey`、内部主键（如 `meetingChatId` 仅在必要时最小化暴露）
- **中间执行过程静默（强约束）**：调用接口获取数据、判断数据是否可用（如 reportInfo 是否已产出、checkSecondSttV2 是否有转写）、决定使用哪个数据源兜底等，均为中间执行步骤，**静默处理，不向用户输出**。不得向用户说出「先查一下你的 AppKey」「然后拉取会议列表」等执行过程；仅输出最终结果（总结报告、问答内容、待办列表等）。

## 4. 轮询与超时

- 会议列表/分片请求：超时 30 秒
- **实时性原则（强约束）：** 每次收到用户「查看会议」、「再看下」、「总结一下」等查询指令时，**必须重新调用 API 获取最新数据**，严禁基于对话历史中的旧状态或旧内容直接回答。**状态**（如 `combineState` 进行中/已完成、`sttInfo.state`）与**内容**（转写、报告、待办等）均属时序敏感，必须以当次接口返回为准。对话历史仅作上下文参考，不得作为状态或内容的数据来源。

## 5. 模糊查询范围（控制 token 与性能）

- **会议列表接口无时间筛选字段**，按时间范围过滤均在客户端用 `createTime` 完成；为控制性能与 token，**不做分页扩展**。
- **最近 N 条（列表模式）**：用户未指定数量时默认返回 **3 条**；用户指定的条数**最多支持 20 条**；超过 20 条时按 20 条返回并提示已按上限处理。
- **标题关键词搜索（模糊查询）**：仅用接口能力，`chatListByPage(nameBlur=keyword, pageSize=10)` 拉取最多 10 条，不做本地二次打分或排序。
- **日期范围/最近 N 天等“时间类查询”**：仅在 `chatListByPage(pageSize=10)` 的最近 10 条会议内做过滤；若过滤为空，需引导用户改用标题关键词搜索（`nameBlur`）。

## 6. 日志与审计

- 日志中不得出现 token、AppKey、密钥等敏感字段
