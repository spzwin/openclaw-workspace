## 五、架构决策

### 当前架构：文档驱动

```
Cron → 读取 meta.json → 检查状态 → 查找下一步 → 发送任务
```

**优点：**
- ✅ 已有实际数据
- ✅ 实现简单
- ✅ 易于调试

**缺点：**
- ❌ 轮询效率低
- ❌ 响应延迟高

### 未来架构：事件驱动（候选）

```
Agent 完成 → 提交通知 → 调度中心 → 派发下一步
```

**优点：**
- ✅ 实时响应
- ✅ 资源高效
- ✅ 扩展性好

**缺点：**
- ❌ 需要迁移数据
- ❌ 开发成本高

**决策：** 短期使用文档驱动，长期评估迁移
